# a-session
  
## プロジェクト概要
a-sessionは、チームでのアカペラ練習をもっと簡単にできるサービスです。メンバー管理、練習スケジュール共有、音源アップロードなど、アカペラチームの活動をサポートします。

## 機能
- ユーザーの新規登録機能(Sign in with googleとAppleIDをサポート)
- ユーザーの退会機能
- ユーザーのログイン機能
- ユーザーのログアウト機能
- 楽譜のアップロード
- 楽譜のダウンロード
- 楽譜の閲覧
- チームでの楽譜共有
- 楽譜音源(MIDI)のアップロード
- 楽譜音源(MIDI)のダウンロード
- 楽譜音源(MIDI)の再生
- 楽譜音源(MIDI)のアシスト演奏
- 楽譜音源(MIDI)のアシスト演奏しながらのリアルタイムレコーディング（録音）
- 楽譜音源(MIDI)のミキサー機能（トラック別ボリューム調整）
- 楽譜音源(MIDI)のミキサー機能（MIDIとレコーディングデータの同時再生）
- 楽譜音源(MIDI)の譜面表示
- 楽譜音源(MIDI)のミキサー機能（MIDIとレコーディングデータの同時再生）のレビュー機能（ユーザー投稿）
- 楽譜音源(MIDI)のミキサー機能（MIDIとレコーディングデータの同時再生）の共有機能
- レコーディングデータのアップロード
- レコーディングデータのダウンロード
- メトロノーム機能
- チーム作成機能
- チーム削除機能
- チームメンバーの追加
- チームメンバーのリーダー設定
- チームメンバーの一覧
- チームメンバーの削除
- 練習スケジュール管理
- 管理者からユーザーへのお知らせ機能
- 管理者機能：ユーザーの一覧取得

- 管理者機能：楽譜の一覧取得

- 管理者機能：楽譜の削除

- 管理者機能：楽譜音源（MIDI)のロックアウト
- 管理者機能：楽譜音源（MIDI)のロックアウト解除
- 管理者機能：楽譜音源（MIDI）の削除
- 管理者機能：チーム一覧の取得
- 管理者機能：チームのロックアウト

## 技術構成
- バックエンド: ASP.NET Web API
- フロントエンド: Flutter
- データベース: PostgreSQL
- ユーザー認証：JWT


## セットアップ手順

### バックエンド (ASP.NET)
1. PostgreSQLをインストールし、起動:
   - Windows: PostgreSQL公式サイトからダウンロードしてインストール
   - デフォルト設定: ユーザー名 `postgres`, パスワード `postgres`
   - ポート: 5432

2. PowerShellで以下を実行:
	```pwsh
	cd backend
	dotnet restore
	dotnet run
	```
   ※ データベースが存在しない場合は自動的に作成され、管理者アカウントが作成されます。

3. APIは http://localhost:5187 で起動します。

### 管理者アカウント
初回起動時に以下の管理者アカウントが自動作成されます：
- **メールアドレス**: supervisor@asession.com
- **パスワード**: Admin123!
- **権限**: 管理者権限あり

### フロントエンド (Flutter)
1. PowerShellで以下を実行:
	```pwsh
	cd frontend
	flutter pub get
	flutter run
	```
2. アプリはエミュレータまたはWebで起動します。
